### Tags / Etiquetas

* Sirve para dar un nombre significativo a un commit concreto
* Una etiqueta concreta no debería ser modificada a lo largo del tiempo
* Lo habitual:
 * 0.1, 0.2, 0.2.1, 1.0, ...
 * v1.0, v1.1, v1.2, ...
 * v1.0.0-alpha, 1.0.0-beta, 2.0.0-rc.1 
 * v20170510, 20170740, 20171128, …
